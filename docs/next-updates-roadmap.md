# Better Fullstack — Current Next Updates Roadmap

> **Updated:** 2026-06-30 after auditing `docs/plans`, schemas, template directories, MCP tools,
> package scripts, smoke/ScaffBench harnesses, and web builder metadata.
>
> **How to read this:** The recommended next update is the smallest product slice that compounds the
> current quality and stack-graph foundation.

## Current P0 Status

### 1. Public Verified-Combination Status — Landed

The first status artifact now exists at `docs/verified-combinations.md`, generated by
`bun run build:verified-combinations`, and the public docs page is available at
`/docs/reference/verified-combinations`. A Shields-compatible badge endpoint is available at
`/api/verified-combinations`.

- It rolls up PR-core smoke, ScaffBench 2, and release-guard evidence.
- It shows checked stacks, commands, ecosystem, validation steps, pass/fail state, and owner area.
- It makes Pass rows the only verified claim; configured/matrix rows remain visible but are not green
  merge claims.
- It starts with curated presets rather than every theoretical combination.
- The docs page renders the current claim from the generated web data module, so it stays in sync
  with the ledger.
- Each evidence surface links to source artifacts, rerun commands, and owner areas for failure
  triage.

Remaining quality depth: keep the claim scoped to rows with current Pass evidence and refresh the
evidence surfaces before widening the guarantee.

### 2. Generated-Project CI Quality — Baseline Landed

The `github-actions` addon now generates more useful CI for the covered stacks.

- Generate ecosystem-aware install/typecheck/build/test workflows for TypeScript, React Native, and
  graph-only Rust/Python/Go/Java/Elixir/.NET projects.
- Keep workflows conservative: run only checks that the selected stack actually defines.
- Add template tests that assert CI output for TypeScript, React Native, Rust, Python, Go, Java,
  Elixir, and .NET presets.

Remaining quality depth: keep adding richer generated-project checks as new ecosystem/template
surfaces land.

### 3. Published-Package Smoke Coverage — Landed

The published-package smoke lane now records package-manager install/scaffold evidence outside the
source checkout at `testing/.published-package/summary.json`.

- `bun run test:published-package` runs `bunx`, `npx`, and `pnpm dlx` against the selected package
  specifier.
- The default package specifier is `create-better-fullstack@latest`; `BFS_PACKAGE_SPECIFIER` or
  `--specifier` can target preview/canary tags.
- The lane verifies that a generated TypeScript/Hono/tRPC/SQLite/Drizzle scaffold produces the
  expected project files for each package manager.
- `bun run build:verified-combinations` includes the package-manager evidence in the generated
  ledger, public docs summary, and Shields-compatible badge count.

## Recommended Next Update

### 1. Harden MCP Stack Updates, Then Promote Them To User-Facing Add

Generic MCP stack updates should become a supported product path, not just an agent interface.

- Keep `bfs_plan_stack_update` / `bfs_apply_stack_update` preview-first semantics.
- Add CLI affordances once the overwrite/conflict model is stable.
- Prioritize additive service updates first: email, observability, search, vector DB, file storage,
  rate limiting, mobile features, and generated CI.
- Keep risky architecture swaps behind explicit review: database, ORM, auth, API, payment provider.

### 2. Agent Plugin Distribution

Better Fullstack already has MCP tools, generated agent docs, and repo-local skills. The next
distribution step is a first-class agent plugin bundle that makes those surfaces installable as one
product.

- Package a plugin manifest with the Better Fullstack MCP server, scaffold/add skills, and install
  metadata for agent clients that support plugins.
- Reuse the existing `bfs_*` MCP flow instead of creating a parallel scaffolding path.
- Include skill triggers for project creation, additive stack updates, and existing-project addon
  workflows.
- Keep generated projects side-effect-light by default: plan first, create with installs disabled,
  then report exact install/test/dev commands.

### 3. AI Chat Example Modernization

The generated AI example should feel like a current product surface, not a bare streaming demo.

- Upgrade the TypeScript AI example surface to AI SDK 7 response helpers across supported backends.
- Prefer shared shadcn-style chat components for React web stacks when the selected UI setup
  supports them.
- Keep non-React and native examples aligned on the same message shape, transport assumptions, and
  env variables.
- Add snapshot and generated-project checks for at least Next, TanStack Start, Hono/Express/Elysia,
  and React Native AI presets.

### 4. Post-Scaffold Upgrade Engine

This is the next major-version moat after stack updates are stable.

- Record scaffold metadata: generator version, template version, stack graph, and selected package
  channel.
- Re-render the intended current template output and diff it against the user's project.
- Produce a reviewable plan instead of overwriting files.
- Add `bfs check` for CI drift detection and `bfs update` for guided patch application.
